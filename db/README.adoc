= db => database

your data — events, profiles, options, queries, conversations, wallnut — persists locally across reloads. there's no user-facing `db` command; storage just happens.

if something goes wrong and you want a clean slate, type `.zzz`. wipes everything (caches, IndexedDB, localStorage, sessionStorage) and reloads.

to disable the offline service worker cache: `.o add cash off`.
