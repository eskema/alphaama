= w => wallnut

zaps (NIP-57), NWC (NIP-47), cashu (NIP-60/61).

zap and nwc are always available. cashu features are gated behind an enable button in the module panel.

== zap

to zap a note or pubkey, type: `.w zap <amount> <pubkey> "<memo>" <event_id>`

* `amount` — sats to send
* `pubkey` — recipient hex pubkey
* `memo` — optional comment (in quotes)
* `event_id` — optional, zaps a specific note instead of just the pubkey
* example: `.w zap 21 abc123... "nice post" def456...`
* the zap button on notes prefills this command with defaults
* defaults: amount from `.o add zap <n>` (default 21), memo from `.o add zap_memo "<text>"` (default `<3`)

== nwc

NWC lets you pay invoices automatically through a remote service.

to set up NWC, type: `.w nwc <connection_string>`

* `connection_string` — a `nostr+walletconnect://` URI from your NWC provider
* the secret is stored encrypted (not in localStorage)

* `.w nwc` — check status
* `.w nwc clear` — remove

when NWC is configured, zaps are paid automatically. if NWC fails (timeout, error), the manual payment dialog is shown as fallback.

== manual payment

when NWC is not configured or fails, a dialog is shown with:

* a link to open the invoice in a lightning app (`lightning:` URI)
* a QR code to scan with a mobile lightning app
* a button to copy the raw bolt11 invoice

== cashu (NIP-60/61)

when disabled, the module shows an enable button. on enable, wallnut syncs your config, proofs, mints, and incoming nutzaps from relays.

=== wallnut setup

* `.w add <mint_url>` — add a mint
* `.w key <privkey>` — set the wallnut private key (offers to publish k:10019 and k:17375)
* `.w import <event_id>` — import wallnut config from kind 17375/37375
* `.w proofs <event_id>` — import proofs from kind 7375

=== minting & melting

* `.w quote <amount>` — request a mint quote (bolt11 to pay)
* `.w mint <quote_id>` — mint proofs after paying the quote
* `.w melt <invoice>` — pay a lightning invoice using cashu proofs

=== tokens

* `.w token <amount> [pubkey] [memo]` — create a sendable cashu token
* `.w receive <token>` — receive a cashu token
* `.w redeem <event_id>` — redeem a nutzap (kind 9321)

=== event creation

* `.k 7375 [mint_url]` — publish proof backup (all mints if no url). deletes old proof events.
* `.k 9321 <amount> <pubkey> [memo] [event_id]` — send a nutzap
* `.k 10019` — publish wallnut discovery (mints + p2pk)
* `.k 17375` — publish wallnut config (encrypted mints + privkey)

=== migration

if you have an old kind-37375 record, a migration button is shown to upgrade it to the current kind-17375 format.
